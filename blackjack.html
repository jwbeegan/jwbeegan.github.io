Blackjack is a demo I created to showcase my skills with Unity. I chose to create a version of blackjack because it removed any design considerations from the workload. This allowed me to focus on creating the most efficient, clear and extendable code possible, while still completing the project over the course of a few weeks. You can find the code for Blackjack on my github here.

Techniques used:

  • State machines to organize and isolate code for various stages of the game.
  • Events to monitor for state changes, without utilizing inefficient update loop checks.
  • Animation control to trigger animation start and finish events.
  • Serialized objects to efficiently store card deck information. Also utilized to create test decks to aid in testing and development.
  • Shader graphs to create cardback and chip textures.
